# MetaVerify - Sistema de Verificação Meta Business

## Visão Geral
SaaS para criação e validação de empresas no ecossistema Meta Business. Trust Infrastructure Layer que automatiza a aprovação de empresas.

## Stack Técnica
- Framework: Next.js 14 com App Router
- DB: PostgreSQL via Prisma ORM
- Auth: NextAuth.js v4 com CredentialsProvider (email/senha)
- Storage: AWS S3 para uploads de documentos
- Charts: Recharts
- UI: Tailwind CSS, Radix UI, Framer Motion
- Idioma: pt-BR

## Estrutura de Páginas
- `/login` - Login
- `/signup` - Cadastro
- `/dashboard` - Dashboard com métricas e gráficos
- `/empresas` - CRUD de empresas (lista, nova, detalhe)
- `/sites-verificacao` - Gerador de sites BMS com preview
- `/trust-score` - Ranking e análise de Trust Score
- `/contas-meta` - Gestão de contas Meta Business
- `/auditoria` - Logs de auditoria

## Schema do Banco
- User (email, name, password, role: ADMIN/FUNCIONARIO)
- Empresa (razaoSocial, nomeFantasia, cnpj, segmento, status, trustScore)
- Documento (tipo, cloudStoragePath, status)
- ContaMeta (nome, metaBusinessId, adAccountId, status)
- SiteVerificacao (dominio, template, conteudoGerado)
- TrustScoreHistorico (score, detalhes)
- AuditLog (acao, descricao, entidade)

## Dados
- Dados fictícios foram REMOVIDOS do banco
- Sistema opera com dados REAIS via Meta Graph API
- Banco limpo: apenas usuários de autenticação mantidos
- Para popular: usar Integração Meta > Importar Dados

## APIs de Integração Real
- `/api/meta-api/test-connection` - Testa token e lista businesses acessíveis
- `/api/meta-api/import` - Importa automaticamente contas, WABAs, números, ad accounts, páginas
- `/api/meta-api/sync` - Sincroniza dados de uma conta específica
- `/api/meta-api/config` - Configuração de credenciais (App ID, Access Token, etc.)

## Fluxo Real
1. Admin configura credenciais em Integração Meta > Configuração
2. Testa conexão (valida token, lista scopes, businesses)
3. Cadastra empresa vinculada ao CNPJ real
4. Importa automaticamente todos os dados da Meta API
5. Cada conta pode ter Access Token individual ou usar o global
